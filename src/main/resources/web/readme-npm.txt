准备环境 : ————————————————————————————————————————————————————————————————————————————————————————————————————

		1:安装 node.js ;
		2:使用cmd ,切换到前端项目开发目录下（本项目是：quickgo-gen\src\main\resources\web）

开发：————————————————————————————————————————————————————————————————————————————————————————————————————————
			
	一(依赖)：
	
		npm install (下载项目依赖包 依赖包已经配置在package.json ， 也可以直接解压提前准备好的node_modules到package.json同级目录);
		
		npm install webpack-dev-server (安装webpackServer，用于本地开发)
		
	二(本地开发测试):
	
		npm run server  (运行webpackServer 用于开发过程中)
		
		#浏览器访问http://localhost:8080/ 可以看到正在开发中的项目
		
	三(打包提交)：
	
		npm run build && gulp  (项目开发完成后打包提交 / 也可以直接使用写好的脚本 z.bat执行此命令  )
	
		npm run dist-server (运行dist文件夹下的项目  ,开发过程不需要 ， 一般用于上线前最后测试。)
		
	
其他：——————————————————————————————————————————————————————————————————————————————————————————————————————————

	html/assets/js/config.js 是项目配置信息；
	html/dashboard/app/route.config 是路由信息；
	webAPP里的built和dist是压缩生成的项目文件，用于线上使用 ；
	
备注：————————————————————————————————————————————————————————————————————————————————————————————————————————————————————

	提交项目无需提交node_modules文件夹，无需提交web下的 dist 和 built 文件夹 
	若使用git命令行提交可以在根目录找到 .gitignore 文件，在最后添加不需要提交的文件；
		如：
			node_modules/
			resources/web/built/
			resources/web/dist/
